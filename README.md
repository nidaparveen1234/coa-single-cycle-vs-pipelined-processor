# COA: Single-Cycle vs Five-Stage Pipelined Processor

This repository contains my Computer Organization and Architecture (COA) project comparing a Single-Cycle Processor and a Five-Stage Pipelined Processor.

## Project Overview
The objective of this project is to understand how instructions are executed in different processor architectures and how pipelining improves CPU performance. The project explores the structural differences between a single-cycle processor and a pipelined processor with five stages.

## Concepts Covered
- Single-cycle processor architecture
- Five-stage pipeline architecture
- Instruction stages: IF, ID, EX, MEM, WB
- Performance comparison between single-cycle and pipelined execution

## Tools Used
- RISC-V simulation using the Ripes processor simulator

## Repository Contents
- **Report/** – Detailed project report including explanation, diagrams, and screenshots
- **Presentations/** – PPT slides used for project presentation
- **Code/** – Source code used in the Ripes simulation

## Learning Outcome
This project helped me understand how pipelining improves processor throughput and how multiple instructions can be executed simultaneously at different stages of the pipeline.
